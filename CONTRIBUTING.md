# Contributing to lenv

Thanks for your interest in contributing.

## Development setup

```bash
git clone https://github.com/AmirhoseinMasoumi/lenv.git
cd Lenv
go build ./...
go test ./...
```

## What to contribute

- Bug fixes with tests
- Cross-platform runtime improvements
- Docs and examples that improve onboarding
- Performance and stability improvements in VM lifecycle code

## Pull request checklist

- Keep changes focused and small
- Add or update tests when behavior changes
- Run `go build ./...` and `go test ./...` before opening PR
- Explain the problem, approach, and validation in PR description

## Need help?

Open an issue with reproduction steps and environment details.
